Nullや空の安全な扱い

Spring + Thymeleaf でよく使う「null や空のときの安全な書き方」を、短いサンプルでまとめています。 画面側で不用意に null 例外を起こさないための“お作法”として押さえておきましょう。

■ Elvis演算子 (?:)

左側が null または空文字のとき、右側のデフォルト値を採用します。

  • value ?: 'デフォルト' の形で「なければこれ」を簡潔に書ける。
  • 画面に「null」や空文字がそのまま出てしまうのを防げる。

■ 安全呼び出し (?.)

左側が null の場合は、その先のメソッド呼び出しをスキップして null を返します。

  • obj?.method() と書くことで、NullPointerException を避けられる。
  • さらに Elvis演算子と組み合わせて「なければ 0」なども簡潔に表現できる。

■ プレーンHTMLのフォールバックテキスト

th:text と要素内のテキストは、どちらが優先されるかを理解しておくとデバッグが楽になります。

  • 通常は th:text が優先され、要素内のテキストは上書きされる。
  • テンプレートを直接ブラウザで開いたときだけ、要素内のテキストが表示される。

■ List の空判定 (#lists.isEmpty)

null か要素数 0 のどちらでも「空」とみなしてくれるユーティリティです。

  • #lists.isEmpty(list) は null も安全に判定可能。
  • 「データなしメッセージ」と「一覧表示」の出し分けに使うのが典型パターン。

■ Map の空判定 (#maps.isEmpty)

Key-Value 形式のデータにも、同じような感覚で使えます。

  • #maps.isEmpty(map) も null 対応済み。
  • 設定値・マスタ情報・在庫情報など、Map で渡すケースの有無チェックに便利。

■ Set の空判定 (#sets.isEmpty)

重複なしコレクションでも、同じパターンで扱えます。

  • #sets.isEmpty(set) で null／空の両方を一括判定。
  • 「選択済みタグ」「所属グループ」など、0件かどうかの分岐に使いやすい。
